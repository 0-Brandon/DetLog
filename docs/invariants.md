# Safety invariants and protection map

The simulator's observer evaluates these properties after every event. It uses
the event history and durable replicas rather than trusting a node's own
`commitIndex` flag.

| ID | Precise property | Protecting implementation path | Primary tests |
| --- | --- | --- | --- |
| I1 Election safety | At most one candidate obtains a majority vote certificate in a term. | [`RaftNode::begin_election` / `on_request_vote_response`](../src/raft.cpp), plus [`register_leader` and the independent observer](../src/cluster.cpp). | `raft_tests`: storage-gated election; `failure_history_tests`: simultaneous election timeouts. |
| I2 Term monotonicity | A node's durable term never decreases and it grants at most one candidate a vote per term. | [`RaftNode::on_request_vote` / `step_down`](../src/raft.cpp) and [WAL semantic validation](../src/wal.cpp). | `raft_tests`: term-zero, durable grant, higher-term stale candidate; `wal_tests`: vote/term regressions. |
| I3 Log matching | If two logs contain the same index and term, their prefixes through that index are identical. | [`RaftNode::on_append_entries`](../src/raft.cpp), atomic WAL suffix replacement, and [`inspect_invariants`](../src/cluster.cpp). | `cluster_tests`: negative Log Matching; `reference_model_tests`: divergent-tail differential repair. |
| I4 Leader completeness | Every leader elected after an entry commits contains that entry. | Vote freshness in [`on_request_vote`](../src/raft.cpp), majority intersection, and the observer's immutable committed ghost prefix. | `cluster_tests`: old leader rejoin; `failure_history_tests`: all three leader-crash boundaries; reference divergent-tail history. |
| I5 Commit certificate | Every newly committed index is justified by durable storage on a majority and the leader's current-term rule. | [`RaftNode::maybe_advance_commit`](../src/raft.cpp) and observer-side durable replica certificates in [`inspect_invariants`](../src/cluster.cpp). | `raft_tests`: current-term-only commit; `cluster_tests`: durable quorum/minority partition; failure history: quorum replication before reply. |
| I6 Committed entries are immutable | No valid log edit removes or changes an entry at or below a known commit watermark. | Follower truncation guard in [`on_append_entries`](../src/raft.cpp), [WAL batch validation](../src/wal.cpp), and the observer ghost prefix. | `wal_tests`: committed truncation rejection; reference divergent-tail repair; randomized property histories. |
| I7 State-machine safety | No two nodes apply different commands at the same index. | Commit/log checks plus the observer's independently rebuilt KV/session state in [`compare_applied_prefix`](../src/cluster.cpp). | `reference_model_tests`: production/reference outcomes; `property_tests`: check after every event. |
| I8 Apply discipline | `lastApplied` increases one at a time and never exceeds `commitIndex`. | Single ordered [`RaftNode::apply_committed`](../src/raft.cpp) path; restart replay uses the same state machine. | `raft_tests`: storage-gated apply; `failure_history_tests`: checked after every event; process restart integration. |
| I9 At-most-once client effect | A `(session, sequence, command digest)` changes the state machine at most once. | [`KvStateMachine::apply`](../src/state_machine.cpp) and cached-session handling in [`RaftNode::on_client_request`](../src/raft.cpp). | `reference_model_tests`: retry/conflict/get/erase differential; `failure_history_tests`: lost-success ambiguous retry; real TCP partition retry. |
| I10 Tail recovery | Discarding an incomplete final WAL frame cannot alter any prior valid frame. | [`scan_wal_bytes`](../src/wal.cpp), checksummed frames, last-valid offset, and durable tail truncation. | `wal_tests`: every final-frame cut and checksum failure; `process_crash_tests`; follower torn-write history. |
| I11 Replay determinism | A fixed executable, configuration, workload, and seed yields the same semantic trace. | Pinned PCG, integer time, total event order, canonical codec, and canonical JSONL in the simulator/cluster adapters. | `cluster_tests`: byte-identical seed replay; property seed/action output; checked-in hashed trace corpus. |
| I12 Resource bounds | A slow or partitioned peer cannot cause a configured queue or uncommitted-log bound to be exceeded. | [`RaftNode::can_append_client`](../src/raft.cpp), bounded simulator/TCP queues, `NodeHostLimits`, and bounded group quarantine. | `raft_tests`: admission/batch bounds; simulator saturation; real TCP partition queue gauges; group-commit quarantine test. |
| I13 Group durability gate | No persistence-dependent send, timer, trace, public consensus status, apply, or client success escapes a safe group before its shared flush succeeds. | `StorageWorker::run_grouped`, operation-tagged quarantine, `complete_group_barrier`, and fail-stop handling in [`node_host.cpp`](../src/node_host.cpp). | `node_host_group_commit_tests`: pre-barrier invisibility, multi-frame grouping, recovery, and default flush-every regression. |

Convergence is not itself a safety oracle. Tests must check the relevant
property throughout the history, including histories that never regain
liveness.
